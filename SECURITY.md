# Security & Bug Reporting

Found a bug or have a security concern? We value your help in making Chord better!

- **Fix it yourself?** Please submit a Pull Request (PR). We love contributions!
- **Can't fix it?** Please file an Issue on GitHub with details on how to reproduce it.

## Philosophy
Chord follows a strict **"No Bloat, Performance First"** philosophy. When reporting a bug or submitting a fix, please ensure:
- **Zero Bloat**: No unnecessary features, dependencies, or complexity are added.
- **Performance First**: CPU and RAM usage are minimized. No performance degradation is allowed.
- **Simplicity**: Code remains clean, maintainable, and technically excellent.

Chord is free and open-source software. Anyone is welcome to contribute and improve the project for everyone!
